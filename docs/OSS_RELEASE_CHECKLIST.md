# OSS Release Checklist

This checklist defines when the Community repository can be published and what must remain private.

## Release profiles

### Portfolio preview

Use this profile when the goal is to show engineering quality, architecture, and a working demo without claiming full product maturity.

Required:

- One-command demo works on a clean machine.
- Unit tests pass.
- No secrets or private-core files are present.
- README clearly labels the project as a prototype / developer preview.
- Public/private boundaries are documented.
- License, notice, third-party notices, contribution, security, and trademark docs are present.

### Public OSS release

Use this profile when the repository is ready to be announced more broadly.

Additional requirements:

- CI runs compile checks, unit tests, readiness checks, and basic secret scanning.
- Example data and plugin manifests are documented.
- Installation instructions are reproducible.
- Demo output is generated by commands rather than committed as stale artifacts.
- The repository does not include the commercial Quality Engine, private evaluation datasets, or private prompt/orchestration logic.

### Product release

Do **not** use the public OSS profile as a product readiness signal. Product release requires a separate gate covering live audio capture, local ASR setup, UX polish, privacy controls, and production support.

## Pre-push ritual

```bash
PYTHONPATH=src python -m unittest discover -s tests -v
PYTHONPATH=src python -m compileall -q src tests
PYTHONPATH=src python -m meeting_agent demo --out-dir /tmp/ama-demo
PYTHONPATH=src python -m meeting_agent release-check --root . --profile public_oss --run-tests --out-md /tmp/release.md
PYTHONPATH=src python -m meeting_agent sbom --root . --out /tmp/sbom.json
```

## Never publish

- Real credentials, API keys, or recorded private meetings.
- Private evaluation datasets.
- Proprietary prompts, model-routing heuristics, or cost-optimization logic.
- Enterprise customer configuration.
- Speaker embeddings or personal biometric data.
- Any OSS code whose license conflicts with the repository license.
